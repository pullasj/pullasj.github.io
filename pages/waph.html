    This Lab covers the foundation of the web. We are using PHP and C for Web applications. and we understand HTTP request and response methods
  • Lab 2: Front-end Web Development
    The lab involves two tasks, each focusing on different aspects of web development.

    In Task 1, I created a basic HTML file with an image, and a form, and incorporated JavaScript to implement various functionalities. This includes displaying the current date/time and logging key presses using inline JavaScript, creating a digital clock using a script tag, implementing show/hide functionality for an email using both inline and external JavaScript, and finally, displaying an analog clock using external JavaScript.

    In Task 2, I delve into Ajax, CSS, jQuery, and Web API integration. added a new HTML element for user input, a button, and a div to perform an Ajax GET request to an echo.php web application. The network connections in the browser will be inspected to illustrate the Ajax request/response process. Additionally, to apply CSS to page using inline, internal, and external styles. jQuery will be introduced to send Ajax GET requests to the echo.php web application when corresponding buttons are clicked. Finally, To utilize jQuery Ajax to fetch and display a random programming joke from the JokeAPI when the page loads and use the fetch() method to call the Agify API with user input, displaying the response results.
  • Hackations
  • Hackathon 1: Cross-site Scripting Attacks and Defenses
    This Hackathon focuses on XSS (Cross-Site Scripting) attacks and aims to raise awareness about web security vulnerabilities. there are two tasks

    Task 1 (Attacking): I need to exploit XSS vulnerabilities in a given URL (http://waph-hackathon.eastus.cloudapp.azure.com/xss/) with six levels of attacking. This involves understanding how XSS attacks work, identifying vulnerabilities in the provided code, and applying various techniques.

    Task 2 (Defending): The second part involves mitigating XSS attacks by implementing secure coding practices. This includes incorporating input validation and sanitization of outputs in the code to prevent malicious scripts from being executed.
